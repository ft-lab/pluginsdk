補足

■引数のvoid *

 void function ( ... void *aux = 0);

Shade Plugin SDKのクラスのメンバ関数のほとんどはvoidのポインタ型（void *aux = 0）を引数としてとりますが、これは今後の関数の拡張を考慮しての引数ですので、実際のコーディングでは引数として考慮する必要はありません。なお、関数のリファレンスではこの引数は省略されています。

■プラグインでの単位

Shadeではユーザの選択によってmm,cm,m,km,inch,foot,mile,yardの単位を切り替えることができますが、Shade内部の単位（native unit）は、ユーザの選択した単位に関係なく常に、mmになっています。プラグインにおいても同様に、mm単位で扱うことになります。ユーザが設定した単位系からShade内部の単位に変換する際には、scene_interface::get_native_to_user_unit(), scene_interface::get_user_to_native_unit() で係数を取得します。

頂点データなどの座標値はローカル座標系の座標値になります。ローカル座標系の座標値をグローバル座標系の座標値に変換する際には、shape_class::get_local_to_world_matrix(), shape_class::get_world_to_local_matrix() で変換マトリクスを取得します。ただし、スキンが設定された頂点については、skin_class::get_skin_world_matrix() から変換マトリクスを取得します。

図形ウインドウで表示されている座標系(ユーザ座標系)は、ろくろ機能やローカル座標モードによって傾きなどの座標変換をされている場合があります。グローバル座標系とユーザ座標系を変換するマトリクスは、scene_interface::get_user_to_world_matrix(), scene_interface::get_world_to_user_matrix() により取得します。
