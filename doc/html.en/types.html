Types of Plugins

Importer Plugins

Derived from the importer_interface class to implements importing scene data from a file. The name of the plugin is added to the Import menu. To the user it appears that the types of files s/he can import from have been added.

Image Importer Plugins

Derived from the image_importer_interface class to implements reading image data from a file. To the user it appears that the types of image files s/he can read from have been added.

Exporter Plugins

Derived from the exporter_interface class to implements exporting scene data to a file. The name of the plugin is added to the Export menu. To the user it appears that the types of files s/he can export to have been added.

Image Exporter Plugins

Derived from the image_importer_interface class to implements writing image data to a file. To the user it appears that the types of image files s/he can write to have been added.

Creator Plugins

Derived from the creator_interface class to implement creating shape objects. The name of the plugin is added to the Create menu. To the user it appears that the types of shapes s/he can create have been added.

Modifier Plugins

Derived from the modifier_interface class to implement shape editing functionality. The name of the plugin is added to the Modify menu. To the user it appears that the types of Modify operations have been added.

Attribute Plugins

Derived from the attribute_interface class to implement creating, diplaying and editing via a dialog box of attributes and attach then to various Shade objects. Other plugins can access those attributes, too.

Effector Plugins

Derived from the attribute_interface to implement post effects to a rendered image. The name of the plugin is added to the Effector menu of the rendering options. To the user it appears that the types of effectors have been added.

Renderer Plugins

Derived from the renderer_interface to implement rendering. The name of the plugin is added to the Rendering Method menu of the rendering options. To the user it appears that rendering methods have been added.

Generic Plugins

Derived from the plugin_interface to implement any functionality. The name of the plugin is added to the Plugin menu which in itself does not imply any specific category.

Browser Plugins

Derived from the browser_interface class to add functionality to the browser. The name of the plugin is added to the browser context menu.

Figure Window Plugins

Derivevd from the figure_interface class to add functionality to the figure window. The name of the plugin is added to the figure window context menu.

Image Box Plugins

Derivevd from the image_interface class to add functionality to image boxes. The name of the plugin is added to all image boxes.

Rendering Image Plugins

Derived from the rendering_image_interface class to add functionality to the rendering image window. The name of the plugin is added to the rendering image context menu.

Window Plugins

Derived from the window_interface class to add new windows. The name of the plugin is added to the View menu. To the user it appears that the types of palette windows have been added. You can also embed windows into the motion window, the image window or the control bar.

Shader Plugins

Derived from the shader_interface to implement shading calculations. The name of the plugins is added to the Mapping Pattern menu of the surface attirubtes. To the user it appears that new mapping patterns have been added.
