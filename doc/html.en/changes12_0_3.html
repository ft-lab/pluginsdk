Changes Between Shade 11 and Shade 12 SDK

*Summary

A list of the changes from Shade 11 to Shade 12 SDK.

*Differences

Header Search Path

SDK is the name of this Plugin SDK

In Shade 11, add the following search path

 SDK/include

Note: Add Boost search path if needed (it should work without it).

In Shade 12, add the following search paths

 SDK/include
 SDK/boost_1_43_0
 SDK/include/openexr-1.6.1
 SDK/include/opengl

Proprocessor (Windows Environment)

In Shade 11,

 SXWINDOWS
 SXWIN32
 DEMO_PLUGIN=0
 NDEBUG
 sxdebug=0
 WIN32
 _WINDOWS
 Windows=1
 PLUGIN=1
 STDCALL=__stdcall
 DLLEXPORT=__declspec(dllexport)
 NOMINMAX
 for=if(true)for
 _CRT_SECURE_NO_DEPRECATE

In Shade 12, remove NOMINMAX preprocessor and add the followings.

 SXPLUGIN=1
 _SECURE_SCL=0
 PLUGIN_EXPORTS
 SXCORE=
 SXMODEL=

Proprocessor (Mac OS X Environment)

See the following xcconfig/config.plugins.xcconfig file,

 ARCHS = x86_64 i386
 MACOSX_DEPLOYMENT_TARGET[arch=x86_64] = 10.6
 MACOSX_DEPLOYMENT_TARGET[arch=i386] = 10.6
 //GCC_VERSION = 4.2
 GCC_FAST_MATH = YES
 GCC_SYMBOLS_PRIVATE_EXTERN = YES
 
 GCC_ENABLE_SSE3_EXTENSIONS = YES
 GCC_ENABLE_SUPPLEMENTAL_SSE3_INSTRUCTIONS = NO
 
 MACROS     = SXMACOSX SXDARWIN SXPOSIX __ASSERTMACROS__SXEXPORT= SXCOREFOUNDATION=1
 SXCOCOA=1 __stdcall= DLLEXPORT= STDCALL= __PROJECT_BUILDER__=1 SXCONFIGURATION=$(CONFIGURATION) SXCORE=
 DEBUGMACROS    =
 RELEASEMACROS   = NDEBUG
 BOOSTPATH    = ../../../../boost_1_43_0
 INCLUDEPATH    = ../../../../include
 
 //OTHER_CFLAGS   = -force_cpusubtype_ALL -ftree-vectorize -fvisibility=hidden
 OTHER_CFLAGS   = -force_cpusubtype_ALL -fno-stack-protector
 OTHER_CPLUSPLUSFLAGS = $(OTHER_CFLAGS) -ffor-scope -Wno-invalid-offsetof
 
 PER_ARCH_CFLAGS_x86_64 = -mfpmath=sse
 PER_ARCH_CFLAGS_i386 = -mfpmath=sse -DSXCARBON=1
 
 CONFIGURATION_BUILD_DIR = ./build/$(CONFIGURATION)
 USER_HEADER_SEARCH_PATHS = $(BOOSTPATH) $(INCLUDEPATH) $(INCLUDEPATH)/openexr-1.6.1/macos $(INCLUDEPATH)/openexr-1.6.1/**

It is recommend to leave this (sample/macosx/xcconfig/config.plugins.xcconfig) file unchanged and include it in your plugin development project.

Depend on where you placed the Plugin SDK, you may need to adjust BOOSTPATH or INCLUDEPATH accordingly.

  • In Xcode 3.2, go to the target project window and under the Build tab, select config.plugins at the bottom right corner to use the preset build settings.
  • Choose GCC 4.2 as the target compiler and make sure the preprocessor macro specified as $ (MACROS) $ (RELEASEMACROS)
  • In Xcode 4, under LLVM GCC 4.2 - Code Generation group, make sure to turn off both (Generate Debug Symbols) and (Symbols Hidden by Default). Otherwise plugin cannot be load correctly.

To avoid the above project settings, try simply use the sample projects provided in this Plugin SDK.

Header Files

In Shade 11, you need to include each individual header files from the SDK,

 #include "shadebuildnumber.h"
 #include "plugininterface.h"
 #include "shadeinterface.h"
 #include "pluginids.h"
 #include "sceneinterface.h"

but in Shade 12, simply include the following sxsdk.cxx

 #include "sxsdk.cxx"

and it will include all the needed header files from the Plugin SDK.

Needed source file in your project

In Shade 11, your project need to include the following source files,

 include/com.cpp
 include/vectors.cpp

but in Shade 12,

 include/sx/core/com.cpp
 include/sx/core/vectors.cpp
 include/sx/core/debug.cpp

the location of these source files are changed.

Changes in interface

To call interface such as plugin_interface, add namespace sxsdk:: before it. For instance, sxsdk::plugin_interface.

Changes in class

To call class such as shape_class, add namespace sxsdk:: before it. For instance, sxsdk::shape_class

Vector and Matrix

 vec2
 vec3
 mat4

From Shade 12,

 sxsdk::vec2
 sxsdk::vec3
 sxsdk::mat4

 sx::vec<float,2> <== vec2
 sx::vec<float,3> <== vec3

enums

Add sxsdk::enums namespace before it

 multipath::parameter::z_depth ==> sxsdk::multipass::parameter::z_depth

sxsdk::rgb_class

In Shade 11, the members of rgb_class are (x, y, z), but from Shade 12, it changed to (red, green, blue) as well as (red, green, blue, alpha) in sxsdk::rgba_class.

Color in rgb_class

Previously, rgb_class col = rgb_class::white, but from Shade 12, sxsdk::vec3 sxsdk::color3::white.

 sxsdk::vec3 vcol = sxsdk::color3::white;
 sxsdk::rgb_class col = sxsdk::rgb_class(vcol.x, vcol.y, vcol.z);

sx::rectangle_class

In Shade 11, the members of sx::rectangle_class are (left, top) - (right, bottom), but from Shade 12, it defined as sx::bounds< sx::vec<int,2> >

So it should read as (min.x, min.y) - (max.x, max.y).
