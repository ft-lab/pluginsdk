Detailed Description

Represents a camera.

Multiple cameras can be set in the scene, but camera_interface represents one camera.

When getting the camera info from scene_interface, camera_class, which has no overhead from generated objects, is recommended.

The following function gets this class directly. It may also be passed as a function argument.

scene_interface::get_camera_interface()

virtual sxsdk::mat4 sxsdk::camera_interface::get_world_to_perspective_matrix ( int  i,
void *  aux = 0 
) [virtual]

The matrix that transforms the the view coordinate system of the camera to the world coordinate system.

The matrix that rotates, moves, and normalizes the world coordinates based on the camera position, direction, and bank. The origin is equal to the eye position, and the eye direction is the Z axis.

Normalizes the coordinates so that they fit in the field of view with -1.0 to 1.0 vertically and horizontally, and going from 0.0 to 1.0 deep. Because coordinates outside these boundaries are outside the field of view, clipping occurs in this coordinate system.
