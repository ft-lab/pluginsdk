*The void * argument

 void function ( ... void *aux = 0);

Most class member functions in Shade Plugin SDK take the pointer void (void *aux = 0) as an argument, but because it is an argument that takes into account the expansion of future functions, it is not necessary to consider void as an argument when you actually code. The void argument is abbreviated in the function reference.

*Units in plugins

In Shade it is possible to change the units into mm, cm, m, km, inches, feet, miles, yards according to the user's choice, but the native units in Shade are always mm regardless of the user input. Likewise, in the plugin, the units are also set to mm

The coordinate value of vertex data will always be in the local coordinate system. When converting from local to global coordinates, get the transformation matrix with shape_class::get_local_to_world_matrix() and shape_class::get_world_to_local_matrix(). However, for vertices with skin settings get the transformation matrix from skin_class::get_skin_world_matrix().

The coordinates displayed in the Figure window (user coordinates) are sometimes transformed by lathe rotation or local coordinate mode bank. Use scene_interface::get_user_to_world_matrix(), scene_interface::get_world_to_user_matrix() to get the transformation matrices for global and user coordinates.
